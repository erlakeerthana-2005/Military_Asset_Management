# Render Deployment Checklist

## Pre-Deployment
- [ ] All code committed to GitHub
- [ ] `requirements.txt` is up to date with all dependencies
- [ ] Environment variables configured in `render.yaml`
- [ ] Database migrations included in `init_db.py`
- [ ] JWT_SECRET_KEY will be auto-generated by Render

## Deployment Steps
- [ ] Go to https://dashboard.render.com
- [ ] Click "New +" â†’ "Blueprint Instance"
- [ ] Connect GitHub account and select `Military_Asset_Management`
- [ ] Review the configuration:
  - Backend service name: `military-asset-backend`
  - Database name: `military-asset-db`
  - Region: Singapore
  - Plan: Free
- [ ] Click "Deploy"
- [ ] Wait for deployment to complete (2-5 minutes)

## Post-Deployment
- [ ] Check application logs for errors
- [ ] Test health endpoint: `GET /api/health`
- [ ] Test login endpoint: `POST /api/auth/login`
- [ ] Verify database connection
- [ ] Get the backend URL from Render dashboard
- [ ] Update frontend `.env.production` with backend URL
- [ ] Redeploy frontend

## Monitoring
- [ ] Check Render logs regularly
- [ ] Monitor database usage
- [ ] Track API error rates
- [ ] Set up alerts for failures (if available)

- Dashboard: https://dashboard.render.com
